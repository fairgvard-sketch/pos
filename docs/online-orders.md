# Онлайн-заказы с сайта (050) — фаза 1 MVP «закажи и забери»

Гость открывает публичную страницу, собирает корзину, оставляет имя и телефон.
Касса получает заявку (realtime + звонок), принимает или отклоняет. Принятый
заказ становится обычным заказом кассы (takeaway, `source='site'`), попадает
в очередь бариста и оплачивается на кассе при получении. Фискальный контур
не тронут: чек рождается в обычном `pay_order`.

## Архитектура

```
Гость (телефон)                Supabase                       Касса (T2)
/order/<location_id>  ──POST──▶ Edge Fn public-order ──rpc──▶ online_orders (staging)
       │                          (service_role)                   │ realtime
       │◀─GET (поллинг 5с)─ Edge Fn public-order ◀─rpc─┐           ▼
       │                                               │      экран /online
/order меню ◀────GET──── Edge Fn public-menu           │      Принять ──▶ accept_online_order
                                                       │            └──▶ place_order (цены из
                                                       └──── статус       каталога, сток 047,
                                                                          очередь бариста)
```

Ключевые решения:

- **Анонимный гость ≠ устройство кассы.** Гость ходит только в Edge Functions
  (anon-ключ как bearer); прямого доступа к таблицам нет. Вся валидация, цены,
  идемпотентность и анти-спам — в `submit_online_order` (SECURITY DEFINER,
  EXECUTE только у service_role).
- **Часы приёма без настроек:** заявка принимается только при открытой смене
  на точке. Смена закрыта → гость видит «Сейчас закрыто».
- **Заявка ≠ заказ.** Снапшот позиций и цен хранится в `online_orders` для
  карточки кассира; настоящий заказ создаётся только при принятии, через
  `place_order` (цены пересчитываются из каталога заново, остатки списываются
  триггерами 047, стоп-лист уважается на этапе заявки).
- **Неоплаченный принятый заказ** (гость не пришёл) закрытие смены аннулирует
  как брошенный counter-заказ (035), остаток вернёт триггер 047.
- **Анти-спам в БД:** ≤3 заявок с одного телефона за 15 минут, ≤30
  необработанных заявок на точку, лимиты длины полей, ≤30 позиций.
- **client_uuid** генерирует страница гостя: ретрай POST не создаёт дубликат,
  он же — «секрет» для поллинга статуса и client_uuid будущего заказа.

## Развёртывание (строго в этом порядке)

Проект: `qgmnxrgtlpyqglwqmsej` (сверить ref с `VITE_SUPABASE_URL`!).

1. **Миграция 050** — SQL Editor Supabase Dashboard, файл
   `supabase/migrations/050_online_orders.sql`.
2. **Edge Functions:**
   ```bash
   supabase functions deploy public-menu --project-ref qgmnxrgtlpyqglwqmsej
   supabase functions deploy public-order --project-ref qgmnxrgtlpyqglwqmsej
   ```
   Секреты не нужны: `SUPABASE_URL`/`SUPABASE_SERVICE_ROLE_KEY` инжектятся сами.
3. **Фронтенд** — пуш в `main` (Vercel). Порядок важен: клиент запрашивает
   `online_orders` и `orders.source` — без миграции запросы сайдбара/очереди
   будут падать (не рушат UI, но сыпят ошибки).

## Поведение кассы

- Новая заявка: звонок + тост на любом экране, счётчик на пункте «Онлайн».
- «Принять» — нужна открытая смена (иначе подсказка). Печатается кухонный
  тикет (если включён в настройках кассы), заказ виден бариста.
- «Отклонить» — с необязательной причиной, гость увидит её на странице.
- «Оплатить» при получении — обычный PaymentSheet (наличные/карта/кошельки,
  смешанная), чек — как всегда («Как выдать чек?» / автопечать).
- «Отменить» принятый (гость передумал) — право `void_order`, остаток вернётся.

## Self-service для любого клиента (051)

Настройки → Обслуживание → «Онлайн-заказы»: тумблер приёма (сервер
отклоняет заявки кодом `disabled`, витрина `public-menu` отдаёт
`accepting` — гостевые страницы прячут оформление заранее), готовая
ссылка `/order/<location_id>` с копированием и печать QR-флаера на
встроенном термопринтере. Владелец новой точки включает онлайн-заказы
сам, без разработчика: завёл меню → распечатал QR → повесил на стойку.

Там же — соцссылки (Instagram / Facebook / отзыв в Google): хранятся в
`locations.settings.online_orders`, `public-menu` отдаёт их в
`location.links`, гостевая страница показывает кнопки в подвале главного
экрана (пустая ссылка — кнопки нет).
